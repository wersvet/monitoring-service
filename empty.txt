теперь определимся с сервисными метриками
A) HTTP API
Метрика	Зачем
Requests per Second (RPS)	сколько запросов обрабатываются в секунду
Error Rate (4xx/5xx)	сколько ошибок происходит (важно и 400 и 500 ошибки должны быть).
Latency P95/P99	“плохие” задержки в системе
Top N slowest paths	какие пути тормозят чаще всего
Traffic by path	где сосредоточена нагрузка
Requests by method	GET vs POST vs others
я так понимаю это нужно в каждый микросервис внедрить. или только в gateway?

Business-oriented HTTP KPIs
эти метрики тоже нужные
Successful logins/min	сколько логинов в минуту
Friend requests / min	количество friend-request
Messages created / min	сколько сообщений отправлено

WebSocket Metrics
Active WS connections	сколько клиентов онлайн
WS connection open/close rate	частота подключения/отключения
WS messages per second	реальная нагрузка по сообщениям
WS error events	сбои при обмене
WS avg latency (опционально)	время доставки сообщений
